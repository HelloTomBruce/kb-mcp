# kb-mcp v0.2 计划

**目标**:Tools 补全 (B) + 模糊匹配 (C) + 语义搜索 (D) 一起做,发 v0.2.0

## 范围

### B. MCP/CLI 工具补全

Store Protocol 已经有 `update`/`delete`/`unlink`/`list`,MCP server 只暴露 4 个。
补全:

| 工具 | 用途 |
|---|---|
| `kb_list(type?, tags?, limit?, offset?)` | 浏览 |
| `kb_update(id, **fields)` | 改文档 |
| `kb_delete(id)` | 软删 |
| `kb_unlink(from, to, rel?)` | 拆边 |
| `kb_backlinks(id)` / `kb_outlinks(id)` | 反向链(可选) |

CLI 同步:`kb update` / `kb delete` / `kb unlink`(list 已有)。

### C. FTS5 trigram 模糊匹配

FTS5 默认 tokenizer (unicode61) 只匹配**精确 token**。问题:
- 搜 `sqlite` 找不到 `sqllite`(拼错)
- 搜 `kb-mcp` 找不到 `kbmcp`(分隔符差异)
- 搜 `fastech` 找不到 `fastech-energy`(词干切分)

方案:加 `trigram` tokenizer(unicode61 + 3-gram),对**短查询**做 OR-of-trigrams 重排序;FTS5 主查询保留原样(性能),trigram 走独立虚表。

实际方案:用 SQLite 内置 `trigram` 扩展?**没有**。需要:
- **方案 1**:新虚表 `docs_fts_trgm` 用 `tokenize='trigram'`(FTS5 自带),写时双写,搜时合并 BM25。
- **方案 2**:在 Python 层把 query 切成 3-gram,直接 MATCH。简单但 N+1 复杂度。

**选方案 1**:加 migration 0002 创建 `docs_fts_trgm`,加 trigger 同步,Store 层 search 改用 `UNION ALL` + BM25 归一化合并。

### D. 语义搜索 (sqlite-vec + 共享 Hermes embedding 配置)

- 加 `sqlite-vec` + `pysqlite3` 依赖(后者给 macOS arm64 用)
- migration 0003 创建 `docs_vec(doc_id TEXT PRIMARY KEY, vec BLOB)`
- **不引入本地模型**,复用 Hermes 的 `auxiliary.embedding` 配置:
  - 读 `~/.hermes/config.yaml` 的 `auxiliary.embedding.{provider, model, base_url, api_key}`
  - 加 `~/.hermes/shared/kb_mcp.yaml` 允许用户覆盖(类似 `KB_MCP_HOME` 环境变量)
- Store 层加 `embedder` 抽象:
  - `HttpEmbedder` — POST `{base_url}/v1/embeddings`,body=`{input, model}`,parse `data[0].embedding`
  - `NullEmbedder` — 降级,无 API key 时返回 `[0.0]*dim`(让 search 退化为 lexical)
- search 加 `mode='semantic'`(纯向量) + `mode='lexical'` 已含,`mode='hybrid'` 含义改为"lexical + fuzzy + semantic"三方合并
- 不做本地推理,避免 100MB+ 模型 + 首次下载慢

**关键收益**:5 个 agent 共享同一份 embedding 配置,你已经在 vision 上用 MiniMax 的 API,加一个 embedding 模型走同 API 即可。

## 关键决策

- **Python sqlite 不支持 extension** → 改用 `pysqlite3` 替换 `sqlite3`(几行代码)
- **embedding 首次下载模型慢** → 启动时延迟加载,显式 `kb embed --rebuild` 命令可重建
- **CLI 增量**:`kb update ID --title ... --body ... --tags ...`
- **D 的可选性**:`sqlite-vec` 设为 `extras_require`,没装就降级(lexical only)
